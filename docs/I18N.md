# 插件多语言支持指南

## 概述

本插件模板已实现完整的多语言（i18n）支持，支持以下 5 种语言：

- **zh-CN**: 简体中文（默认）
- **zh-TW**: 繁体中文
- **en-US**: 英语
- **ja-JP**: 日语
- **th-TH**: 泰语

## 语言切换机制

插件通过 URL 参数自动获取主前端的语言设置。主前端在 iframe 中加载插件时，会自动传递 `lang` 参数：

```
http://localhost:5173/?lang=en-US&theme=light
```

插件在初始化时读取该参数并设置对应语言。如果参数缺失或不在支持列表中，默认使用简体中文（zh-CN）。

### 主前端传递语言参数

主前端通过 `PluginLayout.vue` 自动拼接语言参数：

```typescript
const pluginUrl = computed(() => {
  const baseUrl = pluginManifest.value?.url ?? "";
  if (!baseUrl) return "";
  const separator = baseUrl.includes("?") ? "&" : "?";
  return `${baseUrl}${separator}lang=${appStore.language}&theme=${currentThemeName.value}`;
});
```

### 插件端读取语言参数

插件在 `src/i18n/index.ts` 中读取 URL 参数：

```typescript
function getLanguageFromURL(): string {
  const params = new URLSearchParams(window.location.search)
  const lang = params.get('lang')
  if (lang && ['zh-CN', 'zh-TW', 'en-US', 'ja-JP', 'th-TH'].includes(lang)) {
    return lang
  }
  return 'zh-CN' // 默认简体中文
}
```

## 在组件中使用

### 基本用法

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()
</script>

<template>
  <div>
    <h1>{{ t('sample.title') }}</h1>
    <el-button>{{ t('common.add') }}</el-button>
  </div>
</template>
```

### 在 JavaScript/TypeScript 中使用

```typescript
import { useI18n } from 'vue-i18n'

const { t } = useI18n()

ElMessage.success(t('sample.messages.createSuccess'))
ElMessage.error(t('sample.messages.fetchFailed'))
```

### 在表单验证中使用

```typescript
const { t } = useI18n()

const rules = {
  name: [
    { required: true, message: () => t('sample.messages.nameRequired'), trigger: 'blur' },
  ],
}
```

## 添加新的翻译

1. 在 `src/i18n/locales/zh-CN.ts` 中添加中文翻译键
2. 在其他 4 个语言文件中添加对应翻译
3. 在组件中使用 `t('key.path')` 引用

示例 — 添加一个新的业务模块：

```typescript
// zh-CN.ts
export default {
  // ... 已有模块
  newFeature: {
    title: '新功能',
    description: '这是新功能的描述',
  },
}

// en-US.ts
export default {
  // ... 已有模块
  newFeature: {
    title: 'New Feature',
    description: 'This is the description of the new feature',
  },
}
```

## i18n 配置入口

`src/i18n/index.ts` 的完整配置：

```typescript
import { createI18n } from 'vue-i18n'
import zhCN from './locales/zh-CN'
import zhTW from './locales/zh-TW'
import enUS from './locales/en-US'
import jaJP from './locales/ja-JP'
import thTH from './locales/th-TH'

function getLanguageFromURL(): string {
  const params = new URLSearchParams(window.location.search)
  const lang = params.get('lang')
  if (lang && ['zh-CN', 'zh-TW', 'en-US', 'ja-JP', 'th-TH'].includes(lang)) {
    return lang
  }
  return 'zh-CN'
}

const i18n = createI18n({
  legacy: false,           // 使用 Composition API 模式
  locale: getLanguageFromURL(),
  fallbackLocale: 'zh-CN', // 回退到简体中文
  messages: {
    'zh-CN': zhCN,
    'zh-TW': zhTW,
    'en-US': enUS,
    'ja-JP': jaJP,
    'th-TH': thTH,
  },
})

export default i18n
```

## 注意事项

1. **默认语言**: 如果 URL 中没有 `lang` 参数，默认使用简体中文（zh-CN）
2. **回退机制**: 如果某个翻译键不存在，会自动回退到简体中文
3. **语言代码**: 必须使用完整的语言代码（如 `zh-CN`），不支持简写（如 `zh`）
4. **同步更新**: 添加新功能时，请同时更新所有 5 种语言的翻译文件
5. **结构一致**: 保持所有语言文件的键结构完全一致，避免遗漏

## 维护建议

1. 使用有意义的键名，便于理解和维护（如 `sample.messages.createSuccess`）
2. 对于长文本，考虑拆分成多个键
3. 定期检查是否有遗漏的翻译（可通过对比各语言文件的键来发现）
4. 新增页面或功能时，先在 `zh-CN.ts` 中定义所有键，再逐一翻译到其他语言

# 多语言功能测试指南

## 快速测试

### 1. 独立运行测试

启动开发服务器后，在浏览器中通过 URL 参数切换语言：

```bash
cd frontend
npm run dev
```

然后分别访问以下地址，验证界面语言是否正确切换：

- 简体中文: `http://localhost:5173/?lang=zh-CN`
- 繁体中文: `http://localhost:5173/?lang=zh-TW`
- 英语: `http://localhost:5173/?lang=en-US`
- 日语: `http://localhost:5173/?lang=ja-JP`
- 泰语: `http://localhost:5173/?lang=th-TH`

### 2. 在主前端中测试

1. 启动主前端: `cd web && pnpm dev`
2. 登录主系统
3. 在主前端右上角切换语言
4. 打开你的插件页面
5. 验证插件界面语言与主前端一致

## 预期结果

- 所有界面文本应根据 URL 的 `lang` 参数正确显示对应语言
- 不应存在硬编码的中文文本
- 切换语言后，页面上所有可见文本都应变为目标语言
- 如果某个翻译键缺失，应回退显示简体中文（而非显示翻译键名）

## 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 切换语言无效 | URL 参数名错误 | 确认使用 `lang` 参数，如 `?lang=en-US` |
| 显示翻译键名 | 语言文件中缺少该键 | 检查对应语言文件，补充缺失的翻译 |
| 部分文本未翻译 | 组件中硬编码了文本 | 将硬编码文本替换为 `t('key')` |
| 回退语言不生效 | `fallbackLocale` 配置错误 | 检查 `i18n/index.ts` 中的 `fallbackLocale` 设置 |
